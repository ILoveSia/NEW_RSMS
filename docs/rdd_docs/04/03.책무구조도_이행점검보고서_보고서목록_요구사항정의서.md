# 03. 책무구조도 > 이행점검보고서 > 보고서목록 요구사항정의서

## 1. 개요

### 1.1 화면 개요
- **화면명**: 보고서목록
- **화면ID**: RESP_REPORT_LIST
- **화면경로**: 책무구조도 > 이행점검보고서 > 보고서목록
- **화면설명**: 임원, CEO 보고서 등록 및 조회
- **사용자**: 비로그인자/사용자/관리자

### 1.2 주요 기능
- 이행점검 수행결과 보고서 목록 조회
- 보고서 등록 및 수정
- 보고서 검색 및 필터링
- 보고서 상태 관리
- 개선조치 등록 및 관리

## 2. 화면 구성

### 2.1 화면 레이아웃
```
┌─────────────────────────────────────────────────────────────┐
│ [검색영역]                                                    │
│ - 점검연도, 부점명, 점검상태 검색 조건                        │
│ - 개선조치등록, CEO 보고서 작성, 신규 보고서 작성 버튼         │
├─────────────────────────────────────────────────────────────┤
│ [보고서 목록]                                                │
│ - 이행점검 수행결과 보고 목록 (0건)                          │
│ - 순번, 부서, 구분, 점검명, 점검기간, 보고서번호, 상태,       │
```

## 3. 기능 요구사항

### 3.1 보고서 목록 조회 기능 (F001)
- **기능명**: 보고서 목록 조회
- **설명**: 등록된 이행점검 보고서 목록을 조회
- **입력값**: 점검연도, 부점명, 점검상태
- **처리로직**:
  1. 검색조건에 따른 보고서 필터링
  2. 권한별 조회범위 확인
  3. 페이징 처리
  4. 정렬 기능 제공
- **출력값**: 보고서 목록, 총 건수

### 3.2 보고서 등록 기능 (F002)
- **기능명**: 이행점검 수행결과 보고서 등록
- **설명**: 새로운 보고서를 등록
- **입력값**: 점검회차, 점검기간, 검토내용, 첨부파일
- **처리로직**:
  1. 보고서 등록 팝업 표시
  2. 필수항목 유효성 검증
  3. 보고서번호 자동생성
  4. 첨부파일 업로드 처리
  5. 보고서 정보 저장
- **출력값**: 등록결과, 보고서번호

### 3.3 보고서 수정 기능 (F003)
- **기능명**: 보고서 수정
- **설명**: 기존 보고서 내용을 수정
- **입력값**: 보고서ID, 수정내용
- **처리로직**:
  1. 보고서 상태 확인 (작성중인 경우만 수정 가능)
  2. 수정권한 확인
  3. 수정내용 저장
  4. 수정이력 기록
- **출력값**: 수정결과

### 3.4 보고서 삭제 기능 (F004)
- **기능명**: 보고서 삭제
- **설명**: 작성중인 보고서를 삭제
- **입력값**: 보고서ID
- **처리로직**:
  1. 삭제권한 확인
  2. 보고서 상태 확인 (작성중인 경우만 삭제 가능)
  3. 관련 첨부파일 삭제
  4. 보고서 삭제 처리
- **출력값**: 삭제결과

### 3.5 CEO 보고서 작성 기능 (F005)
- **기능명**: CEO 보고서 작성
- **설명**: CEO 전용 보고서 작성 화면으로 이동
- **입력값**: 점검정보
- **처리로직**:
  1. CEO 보고서 작성 권한 확인
  2. CEO 보고서 작성 화면으로 이동
  3. 기본 정보 설정
- **출력값**: 화면 이동

### 3.6 신규 보고서 작성 기능 (F006)
- **기능명**: 신규 보고서 작성
- **설명**: 일반 보고서 작성 화면으로 이동
- **입력값**: 점검정보
- **처리로직**:
  1. 보고서 작성 권한 확인
  2. 신규 보고서 작성 화면으로 이동
  3. 기본 정보 설정
- **출력값**: 화면 이동

### 3.7 개선조치 등록 기능 (F007)
- **기능명**: 개선조치 등록
- **설명**: 점검결과에 따른 개선조치 사항을 등록 및 관리
- **입력값**: 보고서ID, 개선조치 내용
- **처리로직**:
  1. 개선조치 등록 권한 확인
  2. 점검결과 확인
  3. 개선조치 계획 등록
  4. 담당자 및 완료예정일 설정
- **출력값**: 등록결과

### 3.8 보고서 상태 변경 기능 (F008)
- **기능명**: 보고서 상태 변경
- **설명**: 보고서 작성단계별 상태를 변경
- **입력값**: 보고서ID, 변경할 상태
- **처리로직**:
  1. 상태변경 권한 확인
  2. 현재 상태 검증
  3. 상태변경 조건 확인
  4. 상태 업데이트
  5. 상태변경 이력 기록
- **출력값**: 변경결과

## 4. 비기능 요구사항

### 4.1 성능 요구사항
- 목록 조회 응답시간: 3초 이내
- 보고서 등록 처리시간: 5초 이내
- 파일 업로드 처리시간: 10초 이내 (10MB 기준)
- 동시 사용자 수: 100명

### 4.2 사용성 요구사항
- 반응형 웹 지원
- 브라우저 호환성: Chrome, Edge, Safari 최신버전
- 드래그 앤 드롭 파일 업로드 지원
- 키보드 네비게이션 지원

### 4.3 보안 요구사항
- 사용자별 보고서 접근 권한 제어
- 부서별 데이터 접근 범위 제한
- 파일 업로드 보안 검사
- 보고서 조회/수정 로그 기록

## 5. 제약사항

### 5.1 기술적 제약사항
- 대용량 파일 업로드 제한 (50MB)
- 동시 편집 방지 기능 필요
- 기존 시스템과의 연동 요구

### 5.2 업무적 제약사항
- 보고서 결재 후 수정 불가
- 점검기간 종료 후 일정기간 내 보고서 제출 필수
- 상급자 승인 프로세스 준수

## 8. 파일 관리 요구사항

### 8.1 파일 업로드
- 지원 파일 형식: PDF, DOCX, XLSX, PPTX, HWP
- 최대 파일 크기: 50MB
- 한 번에 최대 10개 파일 업로드
- 바이러스 검사 필수

### 8.2 파일 저장
- 파일명 중복 방지 처리
- 저장 경로 보안 설정
- 백업 및 복구 체계 구축

## 9. 알림 및 워크플로우

### 9.1 알림 기능
- 보고서 제출 시 상급자 알림
- 결재 완료 시 작성자 알림
- 반려 시 사유와 함께 알림
- 개선조치 완료예정일 임박 시 알림

### 9.2 워크플로우
1. 보고서 작성 → 2. 제출 → 3. 검토 → 4. 승인/반려 → 5. 완료
- 각 단계별 권한 확인
- 단계별 상태 변경 이력 관리

## 12. 특이사항

### 12.1 개선조치등록 기능
- 점검결과에서 발견된 문제점에 대한 개선방안을 등록하는 기능으로 추정
- 개선조치 계획, 담당자, 완료예정일, 진행상황 등을 관리
- 개선조치 완료 시 효과성 평가 기능 포함 가능

### 12.2 보고서 유형별 차별화
- CEO 보고서: 전사적 관점의 통합 보고서
- 임원 보고서: 부문별 상세 보고서
- 부서 보고서: 개별 부서의 세부 보고서
